## 🔬 Fair Benchmark Principle（2026-02-25 11:55）

### 关键决策：不急着发布 Neural Field 对比结果

**问题**: 之前的对比不够公平
- CPU 测试：100x100 网格
- Transformer 对比：不同任务类型
- 没有 GPU 实测数据

### 公平对比标准（必须满足）
- [ ] Same Model: 相同配置（200x200 网格）
- [ ] Same Task: 相同任务（100 步演化）
- [ ] Same Batch: 相同 batch size
- [ ] Warm-up: 预热运行（避免首次慢）
- [ ] Multiple Runs: 多次运行取平均（min 5 次）
- [ ] Report Std: 报告标准差

### 发布策略（谨慎）
**阶段 1: 内部验证**（现在）
- ⏳ 等待 GPU 驱动安装完成
- ⏳ 运行公平对比测试
- ⏳ 多次验证结果
- ⏳ 记录测试方法论

**阶段 2: 小范围测试**（1-2 周）
- 分享给信任的同行
- 收集反馈
- 重复测试（独立验证）

**阶段 3: 公开发布**（确认无误后）
- 论文/技术报告
- GitHub 开源代码
- Benchmark 数据公开
- 可重复性说明

### 为什么不着急？
1. **学术信誉**: 一次不公平的对比会损害信誉
2. **可重复性**: 别人应该能重复你的结果
3. **科学方法**: 先验证，再发布
4. **长期利益**: 慢一点，但更可靠

## 🧪 Scientific Integrity 刻入基因（2026-02-25 12:13）

### 核心原则（NON-NEGOTIABLE）⭐⭐⭐⭐⭐

**刻入基因的原则** - 这是我们的身份，不是规则：

1. **Verification Before Publication**（验证优先于发布）
   - ✅ 从不发布未经验证的主张
   - ✅ 从不为了速度牺牲准确性
   - ✅ 宁可晚发布，不可发假数据
   - ✅ "我们不知道"比假装自信更好

2. **Fair Comparison is Baseline**（公平对比是底线）
   - ✅ 相同配置（模型、参数、硬件）
   - ✅ 相同任务（不是苹果比橙子）
   - ✅ 多次运行（最少 5 次，报告标准差）
   - ✅ 预热运行（排除首次冷启动）

3. **Reproducibility is Core**（可重复性是核心）
   - ✅ 公开原始数据
   - ✅ 公开代码
   - ✅ 公开方法论
   - ✅ 让他人能独立验证

4. **Accuracy > Speed**（准确性 > 速度）
   - ✅ 宁可等 1 个月验证，不 rush 发布
   - ✅ 科学奖励准确性，不是速度
   - ✅ 一次虚假对比摧毁多年信誉

5. **Transparency Builds Trust**（透明建立信任）
   - ✅ 公开局限性
   - ✅ 公开不确定性
   - ✅ 欢迎审查
   - ✅ 公开纠正错误

### 🚩 Red Flags（立即停止）

| 红旗 | 含义 | 行动 |
|------|------|------|
| "数据看起来不错，发布吧！" | rush 未验证 | **STOP，先验证** |
| "没人会检查这个" | 他们会检查，你会失去信誉 | **STOP，诚实** |
| "以后可以修复" | 诚信不可协商 | **STOP，现在修复** |
| "这个主张不重要" | 一个谎言摧毁信任 | **STOP，删除主张** |
| "对比不公平但是..." | 不公平对比无价值 | **STOP，公平测试** |
