### 1. WebSocket 监听 vs 轮询

**问题**: "Polymarket API 分为监听和轮询，5 分钟的应该是轮询的，基于监听的 websocket 信号呢？"

**澄清**:

```
数据输入层 (双模式)
│
├── 实时模式 (WebSocket 监听) ⭐
│   ├── 连接：wss://api.polymarket.com/ws
│   ├── 事件：价格更新/订单簿变化/成交
│   ├── 延迟：<100ms
│   └── 用途：
│       ├─ 套利机会检测 (价格偏离瞬间)
│       ├─ 流动性突变监控
│       └─ 方向性交易信号触发
│
└── 轮询模式 (REST API, 每 5 分钟)
    ├── 获取：市场状态/账户信息
    ├── 计算：流动性评分/神经场演化
    ├── 执行：周期性交易决策
    └── 用途：
        ├─ 流动性驱动策略 (5 分钟)
        ├─ 方向性交易 (5 分钟)
        └─ 账户同步
```

**WebSocket 数据流**:

```
Polymarket WebSocket
       ↓
┌──────────────────┐
│ WebSocket Client │
│ (持续监听)       │
└──────────────────┘
       ↓
┌──────────────────┐
│ 实时事件队列     │
│ - price_update   │
│ - orderbook_change│
│ - trade_executed │
└──────────────────┘
       ↓
┌──────────────────────────────────┐
│ 策略模块 (实时响应)              │
├──────────────────────────────────┤
│ 1. 套利检测 (<1 秒)              │
│    ├─ 监听 YES/NO 价格变化        │
│    ├─ 计算 sum_price             │
│    └─ 发现套利 → 立即执行        │
│                                  │
│ 2. 流动性突变监控 (<5 秒)         │
│    ├─ 监听订单簿深度变化         │
│    ├─ 计算流动性评分变化率       │
│    └─ 突变 → 触发流动性策略      │
│                                  │
│ 3. 方向性信号 (<10 秒)            │
│    ├─ 监听大额交易               │
│    ├─ 神经场实时更新             │
│    └─ 方向变化 → 生成信号        │

       ↓
┌──────────────────┐
```

### 2. 资本检查 ($1000)

**问题**: "资本检查 (≥$1000) 本金要大于 1000 吗？"

**澄清**:

```python
# ❌ 错误理解：本金必须 > $1000
if capital < 1000:
    reject_trade()  # 不允许交易

# ✅ 正确理解：保留至少 $1000 作为风险储备
available_for_trading = capital - reserved_capital
if available_for_trading < min_trade_size:
    reject_trade()  # 可用资金不足

# 配置
MIN_CAPITAL_RESERVE = 1000  # 最小保留资本 (风险储备)
MAX_POSITION_SIZE = 0.02    # 单笔最大仓位 (2%)
```

**资本分层**:

```
总资本：$10,000
│
├── 风险储备金：$1,000 (10%) ← 不动用，风险缓冲
│
└── 可用交易资本：$9,000 (90%)
    │
    ├── 已用资本：$510 (5.7%)
    │   ├─ 持仓 1: $170
    │   ├─ 持仓 2: $170
    │   └─ 持仓 3: $170
    │
    └── 可用资本：$8,490 (94.3%)
        └── 单笔最大交易：$180 (2% of $9,000)
```

**风控逻辑**:

```python
def risk_check(signal, account):
    # 1. 保留资本检查
    reserved = account.capital * 0.10  # 10% 储备
    available = account.capital - reserved
    
    if available < min_trade_size:
        logger.warning("可用资本不足 (扣除储备金后)")
        return False
    
    # 2. 仓位检查
    if len(account.positions) >= max_positions:
        logger.warning("仓位已满")
        return False
    
    # 3. 单笔风险检查
    position_size = available * max_position_pct  # 2%
    if position_size < min_trade_size:
        logger.warning("单笔交易规模过小")
        return False
    
    return True
```

### 3. 策略权重动态调整

**问题**: "4 大策略权重通过实时数据的模拟进行修正"

**方案**:

```python
class DynamicWeightAllocator:
    """动态权重分配器"""
    
    def __init__(self):
        self.base_weights = {
            'liquidity': 0.50,
            'arbitrage': 0.30,
            'directional': 0.20
        }
        self.performance_history = []
    
    def update_weights(self, performance_data: Dict):
        """根据历史表现动态调整权重"""
        
        # 计算各策略夏普比率
        sharpe_ratios = {
            'liquidity': self.calculate_sharpe('liquidity'),
            'arbitrage': self.calculate_sharpe('arbitrage'),
            'directional': self.calculate_sharpe('directional')
        }
        
        # 根据夏普比率分配权重
        total_sharpe = sum(sharpe_ratios.values())
        if total_sharpe > 0:
            for strategy in self.base_weights:
                # 新权重 = 基础权重 * 0.5 + 表现权重 * 0.5
                performance_weight = sharpe_ratios[strategy] / total_sharpe
                self.weights[strategy] = (
                    self.base_weights[strategy] * 0.5 +
                    performance_weight * 0.5
                )
        
        # 归一化
        total = sum(self.weights.values())
        for strategy in self.weights:
            self.weights[strategy] /= total
    
    def get_weights(self) -> Dict:
        return self.weights
```

**权重调整规则**:

| 条件 | 调整 | 说明 |
|------|------|------|
| 套利胜率 > 80% | 套利权重 +5% | 套利表现好 |
| 流动性回撤 < 5% | 流动性权重 +10% | 流动性稳定 |
| 方向性连续亏损 3 笔 | 方向性权重 -10% | 方向性表现差 |
| NLP 新闻准确率高 | NLP 增强系数 +0.05 | 新闻预测准 |
