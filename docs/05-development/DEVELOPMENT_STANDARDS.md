# NeuralFieldNet 开发规范

**版本**: v1.0  
**生效日期**: 2026-02-26  
**原则**: 无文档不开发

---

## 📜 核心原则

### 1. 无文档不开发 (No Docs, No Code)

```
❌ 错误流程:
需求 → 写代码 → 测试 → 部署 → (忘记写文档)

✅ 正确流程:
需求 → 写文档 → 评审文档 → 写代码 → 测试 → 部署 → 更新文档
```

### 2. 文档先行 (Docs First)

在写任何代码之前，必须先完成:
- [ ] 需求文档
- [ ] 设计文档
- [ ] API 文档 (如有)
- [ ] 测试计划

### 3. 文档即代码 (Docs as Code)

- 文档使用 Markdown 格式
- 文档纳入版本控制 (Git)
- 文档与代码同等审查

## 📝 文档编写规范

### Markdown 规范

```markdown
# 一级标题

## 二级标题

### 三级标题

- 无序列表
- 无序列表

1. 有序列表
2. 有序列表

**粗体**  *斜体*  `代码`

```python
# 代码块
def hello():
    print("Hello, World!")
```

| 表格 | 示例 |
|------|------|
| 单元格 | 内容 |

> 引用文本

[链接](https://example.com)

![图片](image.png)
```

### 命名规范

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| **策略文档** | STRATEGY_NAME.md | `LIQUIDITY_STRATEGY.md` |
| **API 文档** | API_NAME.md | `POLYMARKET_API.md` |
| **设计文档** | DES-XXX.md | `DES-001-liquidity-bot.md` |
| **需求文档** | REQ-XXX.md | `REQ-001-arbitrage.md` |
| **会议记录** | YYYY-MM-DD-meeting.md | `2026-02-26-team-meeting.md` |

## ✅ 合规检查

### 定期检查

- **每周**: 检查新增代码是否有文档
- **每月**: 审查文档完整性
- **每季度**: 更新过时文档

### 违规处理

| 违规类型 | 处理措施 |
|---------|---------|
| 无文档开发 | 代码回滚 + 补文档 |
| 文档不完整 | 暂停开发 + 补全文档 |
| 文档过时 | 标记 + 限期更新 |
