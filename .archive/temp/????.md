# 养老模式完整执行流程

## 核心理念
**稳、慢、少、停得住** - 以长期稳定为核心，杜绝一切可能导致风险的激进行为。

## 系统特征
- **行为稳定** - 永远保持一致的行为模式
- **极低频次** - 严格限制操作频率和数量
- **宁可错过** - 优先考虑安全而非收益
- **一异常就停** - 遇到问题立即停止运作

## 系统架构
```
┌──────────────────┐
│ Market Data     │  (盘口/波动/流动性)
└────────┬─────────┘
         ↓
┌──────────────────┐
│ State Builder   │  (整理成结构化输入)
└────────┬─────────┘
         ↓
┌──────────────────┐
│ Prompt Layer    │  (养老锁死Prompt)
└────────┬─────────┘
         ↓
┌──────────────────┐
│ Qwen2.5:3B      │  Planner 输出一个词
└────────┬─────────┘
         ↓
┌──────────────────┐
│ Decision Filter │  (只接受3种输出)
└────────┬─────────┘
         ↓
┌──────────────────┐
│ Risk Circuit    │  (硬性风险熔断层)
│ Breaker         │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ Executor        │  执行 or 不执行
└──────────────────┘
```

## 各层功能详解

### ① Market Data
- 外部世界数据输入
- 模型只看到被整理后的状态

### ② State Builder
- 将原始数据压缩成结构化状态
- 示例输出：`volatility: low`, `liquidity: high`, `spread: narrow`, `system_status: normal`
- 这是关键安全层

### ③ Prompt Layer（锁死的养老Prompt）
- 决定模型"性格"：默认SKIP、不冒险、不优化、不学习
- 这层决定长期稳定性

### ④ Qwen Planner（无tool）
- 仅作为确定性分类器，不是策略生成器
- 只输出：`SKIP` 或其他两个词

### ⑤ Decision Filter
- 强制校验：如果输出不是预设的3个词 → 强制SKIP
- 这是"保险丝"

### ⑥ Risk Circuit Breaker（硬性风险熔断层）
- 不相信模型，即使输出ALLOW也会再次检查
- 实现"模型可以出错，但系统不能出错"

### ⑦ Executor
- 只有在ALLOW_SMALL_TRADE时才会真正下单
- 否则什么都不做

## 关键参数冻结
### 1. Planner 冻结
- `temperature = 0.0` - 消除随机性
- `default_action = SKIP` - 默认跳过
- `max_calls_per_min = 3` - 极低频调用

### 2. 执行行为冻结
- `min_interval_sec = 2.5` - 永远慢速
- `max_daily_actions = 40` - 严格上限
- `order_size = 50` - 固定小额

### 3. 策略结构冻结
- 仅保留保守策略(S1)
- 关闭所有探索和机会策略
- 禁用自适应学习功能

### 4. 安全阈值冻结
- 更敏感的异常检测
- 更长的暂停时间(10分钟)
- 严格的风险控制

## 决策分布
- **SKIP**: 70-90% - 大多数情况下跳过
- **OBSERVE**: 10-25% - 观察为主
- **ALLOW_SMALL_TRADE**: <10% - 极少执行

## 自适应缩壳机制

### Market Stress Score (MSS) 计算
```
MSS = volatility_score (0–3) +
      spread_score (0–2) +
      liquidity_score (0–2) +
      recent_drawdown_score (0–3)
```

### 根据MSS自动调节
- **MSS ≤ 2**: 正常模式
  - max_position_size_pct: 0.7%
  - max_trades_per_day: 3
- **MSS 3–5**: 保守模式
  - max_position_size_pct: 0.5%
  - max_trades_per_day: 2
- **MSS 6–7**: 非常保守模式
  - max_position_size_pct: 0.3%
  - max_trades_per_day: 1
- **MSS ≥ 8**: SKIP_ONLY模式（完全停止）
  - 无论模型输出什么 → 强制 SKIP

## 完整执行逻辑
```
IF Planner == ALLOW_SMALL_TRADE
AND 未触发任何熔断
AND 未超资金限制
AND 未超频率限制
THEN 执行
ELSE SKIP
```

## 监控指标
仅关注系统健康状态：
- 容器正常运行
- 错误数为0
- 行为符合预期模式
- 异常时及时停止

## 重要原则
1. **绝不优化** - 任何改动都可能破坏稳定性
2. **不手痒** - 不因短期收益而改变策略
3. **不证明自己** - 避免"展示能力"的冲动
4. **长期静默** - 连续不操作是健康的标志

## 核心哲学
> "系统最危险的阶段不是激进，而是赚到一点后开始手痒"

养老模式的成功在于：**不靠聪明、不靠速度、不靠技巧，靠的是时间和不犯错**。这是自动系统能够长期存活的唯一方式。
