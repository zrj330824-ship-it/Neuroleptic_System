## 🔍 故障排除

### 问题 1: 还是没有成交

**可能原因**:
- 市场流动性不足
- 阈值还是太高
- API 连接问题

**解决方案**:
```bash
# 1. 检查日志
tail -f /root/polymarket_quant_fund/logs/trading.log

# 2. 查看是否有信号
grep "套利机会" logs/trading.log

# 3. 进一步降低阈值到 0.2%
sed -i 's/min_arbitrage_threshold: 0.0025/min_arbitrage_threshold: 0.002/' config.json
```

### 问题 2: 成交太多（>20 笔/小时）

**可能原因**:
- 阈值太低
- 并发太高

**解决方案**:
```bash
# 提高阈值到 0.3%
sed -i 's/min_arbitrage_threshold: 0.0025/min_arbitrage_threshold: 0.003/' config.json

# 减少并发到 5 个
sed -i 's/max_positions_concurrent: 8/max_positions_concurrent: 5/' config.json
```

### 问题 3: 系统崩溃

**解决方案**:
```bash
# 1. 查看错误日志
tail -100 /root/polymarket_quant_fund/logs/websocket.log

# 2. 重启系统
cd /root/polymarket_quant_fund
pkill -f websocket_client.py
nohup python3 websocket_client.py > logs/websocket.log 2>&1 &

# 3. 设置自动重启（已有 system_guard.sh）
```
